# Self-Hosting BookmarX

BookmarX is currently designed for self-hosting. Each deployment uses the owner's X Developer app credentials and AI provider key.

## Requirements

- Docker and Docker Compose
- X Developer app with OAuth 2.0 enabled
- AI provider key compatible with the configured models

## Quick Start

Copy the example environment file:

```bash
cp .env.example .env
```

Edit `.env` and set at least:

```env
ENCRYPTION_KEY=
X_CLIENT_ID=
X_CLIENT_SECRET=
X_REDIRECT_URI=http://localhost:4000/auth/x/callback
AI_API_KEY=
```

Start the stack:

```bash
docker compose up -d
```

Run migrations:

```bash
docker compose exec backend bun run db:migrate
```

Open the app:

```txt
http://localhost:3000
```

Check setup status:

```txt
http://localhost:4000/setup/status
```

## Services

- `frontend`: Next.js app on port `3000`
- `backend`: Hono API on port `4000`
- `worker`: background sync and AI jobs
- `postgres`: PostgreSQL with `pgvector`
- `redis`: BullMQ queue backend

## Common Commands

```bash
docker compose logs -f backend
docker compose logs -f worker
docker compose restart backend worker
docker compose down
```

To remove all local data:

```bash
docker compose down -v
```

## Notes

- Manual X bookmark sync is the default flow.
- AI embeddings are generated by the worker after sync.
- The app stores encrypted X OAuth tokens in Postgres.
- Keep `ENCRYPTION_KEY` stable. Changing it makes stored tokens unreadable.
